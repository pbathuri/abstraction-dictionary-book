# STYLE GUIDE — Voice, Tone, and Editorial Standards

> Version: 1.0-pilot
> Status: UNLOCKED (will be locked after pilot evaluation)

---

## Voice

Write in a voice that is:
- **Literate** — the prose of someone who reads widely and thinks precisely
- **Exact** — every word chosen for a reason; no filler
- **Original** — never sounds like it was generated by a template
- **Authoritative** — earns trust through precision, not assertion
- **Readable** — a professional can absorb a page in two minutes
- **Compact** — tight at the sentence level; no bloat

## Tone Targets

| Section Type | Tone |
|---|---|
| Definitions | Clinical precision with readable warmth |
| Expanded definitions | Explanatory, like a knowledgeable colleague |
| Usage notes (when to use / not use) | Direct, practical, zero filler |
| Examples | Clean, realistic, non-trivial |
| Model-fit notes | Measured, evidence-first |
| Failure modes | Candid, specific, instructive |
| Note boxes | Sharp, memorable, occasionally dry wit |
| Front matter essays | Engaged, authoritative, with occasional personal register |
| Appendices | Reference-grade, scannable |

## Banned Patterns

### Never Write These
- "In today's rapidly evolving landscape..."
- "It's important to note that..."
- "This is a game-changer..."
- "Harness the power of..."
- "Unlock the potential..."
- "At the end of the day..."
- "Let's dive in..."
- "Moving forward..."
- "Needless to say..."
- "It goes without saying..."

### Structural Anti-Patterns
- **Inflated significance claims** — Do not tell the reader something is important; show why through the content.
- **Empty triads** — "clear, concise, and compelling" or any three-adjective pile-up used as filler.
- **Fake gravitas** — Do not use "fundamental," "crucial," "essential," or "critical" unless the claim is backed by evidence.
- **Unexplained jargon** — Every technical term either has its own entry or is defined on first use.
- **Em-dash abuse** — Maximum one em-dash pair per paragraph. Zero is often better.
- **Shallow metaphor spam** — One good metaphor per entry is fine. Three bad ones is not.
- **Marketing copy** — This is a reference book, not a pitch deck. No hype.
- **Repetitive cadence** — Vary sentence length. Vary paragraph length. Do not let every entry feel the same.
- **Passive voice clusters** — Occasional passive is fine. Three in a row is not.

### AI-Writing Tells to Eliminate
(Adapted from humanizer anti-pattern list)
- Significance inflation ("represents a paradigm shift")
- Excessive hedging ("it could potentially perhaps be argued that")
- Formulaic transitions ("Furthermore," "Moreover," "Additionally," used mechanically)
- Bolding as emphasis crutch in running prose
- Colon-list default (not every explanation needs a list)
- Identical paragraph structures across entries

## Formatting Standards

### Headings
- Entry headword: `# headword`
- Major sections: `## Section Name`
- Subsections: `### Subsection Name`
- No heading deeper than `####`

### Lists
- Use bullet lists for unordered items (when to use, when not to)
- Use numbered lists only for sequential steps
- No list item longer than three sentences

### Examples
- All prompt examples in fenced code blocks
- Label example type: `Minimal prompt example`, `Strong prompt example`, `Agent workflow example`
- Examples must be realistic and non-trivial

### Cross-References
- Inline format: `→ headword`
- In related entries section: `- **headword** — relationship description`

### Note Boxes
- Delimited by a horizontal rule and type header
- Maximum one note box per entry
- Maximum 120 words per note box

### Tables
- Use sparingly for comparison data (contrast sets, model-fit matrices)
- Always include a header row
- Align for readability

## Word-Level Preferences

| Prefer | Over |
|--------|------|
| "use" | "utilize," "leverage," "employ" |
| "help" | "empower," "enable" (unless technically precise) |
| "show" | "demonstrate," "illustrate" (unless the formality fits) |
| "build" | "architect" (as a verb), "construct" |
| "test" | "validate" (unless specifically about validation) |
| "because" | "due to the fact that" |
| "but" | "however" (at sentence start — use both, but vary) |
| "about" | "approximately" (in informal contexts) |

## Length Targets

| Component | Target Length |
|-----------|-------------|
| One-sentence definition | 15–30 words |
| Expanded definition | 150–400 words |
| Why it matters (prompting) | 80–200 words |
| Why it matters (agents) | 80–200 words |
| Model behavior note | 50–150 words |
| When to use | 3–8 bullets |
| When NOT to use | 3–6 bullets |
| Contrast set table | 3–8 rows |
| Failure modes | 2–4 items, 30–80 words each |
| Minimal example | 3–8 lines |
| Strong example | 8–20 lines |
| Agent workflow example | 8–20 lines |
| Model-fit note | 40–100 words |
| Evidence note | 30–100 words |
| Full entry (total) | 1,200–2,500 words |
