# BOOK BIBLE — The Abstraction Dictionary

**Author: Kastle Light**

## What This Book Is

An interactive reference book that treats natural language as the most powerful programming interface for AI systems. It provides seventy-five named abstractions — each with specific words to use, real project examples, before-and-after prompt comparisons, paste-in exercises, and experiment data from 11,400 prompt variations.

It reads like a smart colleague at your whiteboard, not like an encyclopedia. Each entry is built to be used, not just read.

## What This Book Is Not

- Not a knockoff of Oxford, Roget, or The Well-Spoken Thesaurus
- Not a prompt cookbook or tips-and-tricks collection
- Not a marketing document for any specific AI product
- Not a fiction writing guide adapted for tech
- Not a survey of every possible prompt pattern

## Core Thesis

Natural language has become executable. When you write a prompt, you are programming. When you design agent instructions, you are writing system architecture. The quality of the language determines the quality of the output. This book gives readers the vocabulary, the patterns, and the judgment to write that language well.

## Target Reader

1. **Prompt engineers** who want precision beyond trial-and-error
2. **AI application developers** building agent workflows
3. **Technical writers and editors** who create AI-facing content
4. **Researchers** studying human-AI interaction patterns
5. **Power users** who want to stop fighting their tools

## Book Architecture

### Part I: How to Read This Book (30–50 pages)
- Why natural language now functions like programming
- The difference between words, abstractions, constraints, and executable instructions
- How prompting differs from context engineering
- How agentic workflow language differs from single-turn prompting
- Style, tone, role, rubric, and verification
- Common prompting failure modes

### Part II: The Abstraction Dictionary (260–300 pages)
- Main entries organized by hybrid taxonomy (alphabetical within thematic families)
- Each entry is both a standalone reference and a readable section
- Seven entry families: Core Abstractions, Instructional Actions, Context Architecture, Agent Workflows, Quality Control, Tone/Style, Failure Modes

### Part III: Applied Patterns (40–50 pages)
- Composing prompts from abstractions
- Multi-step planning patterns
- Retrieval, critique, extraction, evaluation prompts
- Agent instruction design
- Delegation and handoff language
- Prompt anti-patterns

### Part IV: Reference Appendices (30–40 pages)
- Model fit matrix
- Capability tiers
- Syntax templates
- Workflow phrasebook
- Glossary of prompt/agent terms
- Bibliography and source policy

## Entry Families

1. **Core Abstraction** — specificity, framing, decomposition, hierarchy, grounding
2. **Instructional Action** — constrain, compare, rank, justify, critique, synthesize
3. **Context Architecture** — context windowing, retrieval scaffolding, memory cueing, source anchoring
4. **Agent Workflow** — delegation, handoff, verification loop, watchdog, planner-executor split
5. **Quality Control** — audit trail, falsifiability, rubric, test harness, regression check
6. **Tone/Style** — register, warmth, authority, terseness, narrative glue
7. **Failure Mode** — vagueness, hallucination bait, overcompression, underspecification, prompt drift

## Editorial DNA

Borrow these principles at the level of function, never wording:
- Dictionary-grade clarity
- Thesaurus-grade contrast
- Style-guide-grade usage notes
- Rhetoric-guide-grade explanation
- Workbook-grade applied examples
- Software-manual-grade reproducibility

## Note Box Policy

- No more than 20–30% of entries carry a note box
- Allowed types: Which Word?, Workflow Note, Model Note, Common Trap, Upgrade This Prompt, Fun Aside (very sparse)
- Tone: sharp, memorable, lightly witty, never goofy
- Icons: sparse, tasteful, no emoji clutter

## Model-Fit Note Policy

- Never invent parameter counts
- Public parameter counts cited with source
- Non-public models classified by tier: small open, midsize open, large open, frontier proprietary, reasoning-specialized, code-specialized
- Empirical notes only when evidence-backed

## What Counts as Original

- Every definition is written from scratch, not paraphrased from existing dictionaries
- Every example is constructed for this book's specific domain
- The taxonomy is our own organizational system
- The entry schema is designed for LLM/agent contexts, not adapted from linguistics
- All prose is original unless explicitly quoted with attribution
